# 出力フォーマット仕様

このファイルは `agent-article-to-markdown` スキルが出力する Markdown の正規フォーマットを定義する。
スクリプト (`extract_article.py` / `extract_xcom.py`) はこのフォーマットに準拠して出力する。

## 構造

```
---
{YAML frontmatter}
---

# {タイトル}

{本文}
```

## Frontmatter フィールド

```yaml
---
title: "記事タイトル"           # 必須
source_url: "https://..."       # 必須: 元URL
author: "著者名"                # 任意: 取得できない場合は省略
published_at: "YYYY-MM-DD"      # 任意: 取得できない場合は省略
retrieved_at: "YYYY-MM-DDTHH:MM:SS+09:00"  # 必須: 取得日時 ISO 8601
site: "example.com"             # 必須: ドメイン名
content_type: "article"         # 必須: article | tweet | thread
---
```

## content_type 定義

| 値 | 説明 |
|---|---|
| `article` | 一般的なWebページの記事 |
| `tweet` | x.com の単一ツイート |
| `thread` | x.com のスレッド（複数ツイートの連続） |

## 本文変換ルール

- 見出し (h1-h6): `#` ~ `######` に変換
- 段落: 空行区切り
- リスト: `-` (箇条書き) / `1.` (番号付き)
- 引用: `>` プレフィックス
- コードブロック: バッククォート3つで囲む
- 画像: 画像リンクは出力せず、OCR で読み取ったテキスト説明に置換
- リンク: `[text](url)`
- スレッドのツイート区切り: `---` (水平線)

## ファイル名規則

- 出力先: `agent-articles/{sanitized_title}.md`
- サニタイズ: `/\:*?"<>|` を除去、空白を `-` に変換
- 最大100文字に切り詰め

## 画像OCR説明フォーマット

画像リンクは出力せず、OCRで読み取った内容をテキスト説明に置換する:

```
> **[図]** 2024年1月〜12月の売上推移。1月: 100万円、6月: 250万円、12月: 400万円。
```

OCRでテキストが読み取れず、alt テキストがある場合:

```
> **[図]** 製品デモ画面
```

テキストも alt も無い場合は画像の痕跡を残さず除去する。

## 設計原則

1. **内容非改変**: 元記事の内容は要約・編集しない
2. **構造保存**: 見出し階層・リスト構造を忠実に再現
3. **LLM最適化**: frontmatter でメタデータを構造的に提供し、本文は余計な装飾を排除
4. **パース容易**: 他のスキルが YAML パースでメタデータを利用可能
5. **画像内容の可読化**: 画像内テキストをOCRで抽出し、LLMが画像なしでも内容を把握可能にする
